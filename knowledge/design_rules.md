# 設計規範知識庫

> 可透過 `python cli.py update-knowledge` 更新內容。

## 通用 UI 設計規則

### 字型
- 標題：16-24px，粗體
- 內文：14-16px，Regular
- 說明文字：12-14px，淺色
- 行距：1.4-1.6 倍

### 間距
- 元素間最小間距：8px
- 區塊間距：16-24px
- 邊距（Padding）：16px
- 使用 4px 或 8px 網格系統

### 色彩
- 主色不超過 3 種
- 背景與文字對比度 ≥ 4.5:1（WCAG AA）
- 錯誤色：紅色系
- 成功色：綠色系
- 警告色：橘/黃色系

### 按鈕
- 最小點擊區域：44 × 44 px
- 主要按鈕（Primary）：填滿色 + 白字
- 次要按鈕（Secondary）：邊框 + 主色字
- 禁用狀態：降低透明度至 40%

### 圖示
- 風格統一（線性 or 填滿，不混用）
- 最小尺寸：24 × 24 px
- 搭配文字標籤輔助理解

### 卡片/容器
- 圓角：8-16px
- 陰影：輕微（0 2px 8px rgba(0,0,0,0.08)）
- 背景色與頁面背景有區分

## 遊戲 UI 特殊規則

### 介面佈局
- HUD 不遮擋主要遊戲畫面
- 重要資訊（血量、金幣）放在固定位置
- 彈窗不超過螢幕 80%

### 動效
- 按鈕回饋動畫 ≤ 200ms
- 頁面切換動畫 ≤ 300ms
- 避免頻繁閃爍

### 文字
- 遊戲內文字需有描邊或陰影增加可讀性
- 重要數值使用大號字體突出

## 常見問題清單

| 問題 | 影響 | 解法 |
|------|------|------|
| 按鈕太小 | 誤觸、挫折感 | 增加到 44px+ |
| 對比度不足 | 看不清文字 | 加深文字 or 淡化背景 |
| 資訊過載 | 使用者迷失 | 分層、摺疊次要資訊 |
| 風格不一致 | 不專業感 | 建立 Design Token |
| 無引導視線 | 不知道看哪裡 | 增加視覺層級差異 |
| 留白不足 | 擁擠壓迫感 | 增加 padding/margin |
